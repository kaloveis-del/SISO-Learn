# 요구사항 문서: SISO-Learn — kids-ai-study-app

> 플랫폼: iPadOS 16.0+ | 언어: Swift 5.9+ | UI: SwiftUI

---

## 소개

SISO-Learn은 중학교 2학년과 초등학교 5학년 두 자녀를 위한 iPad 기반 AI 인터랙티브 학습 앱이다. 강아지 캐릭터 **쿠키(Cookie)** 🐶가 AI 선생님 역할을 맡아 대화 형식으로 학습을 이끌어간다. 단순한 입력-출력 방식이 아닌, 쿠키가 먼저 말을 걸고 질문하고 반응하는 쌍방향 학습 경험을 제공한다.

---

## 용어 정의

| 용어 | 정의 |
|------|------|
| 쿠키 | 앱의 AI 선생님 역할을 하는 강아지 캐릭터. 모든 학습 메시지를 쿠키의 말투로 전달한다 |
| 학습 세션 | 과목·난이도를 선택하고 Quiz를 풀어 완료하는 하나의 학습 단위 |
| Quiz | 쿠키가 출제하는 주관식 서술형 문제 |
| 힌트 | 정답을 직접 알려주지 않는 단계적 도움말 (최대 3단계) |
| 스트릭 | 연속으로 학습한 일수 |
| 성취 배지 | 과목별 정답률 80% 이상 달성 시 획득하는 보상 |
| GradeLevel | 학년 수준 (초등학교 5학년 / 중학교 2학년) |
| AITutorProtocol | AI 제공자를 교체할 수 있도록 추상화한 인터페이스 |

## 요구사항

### 요구사항 1: 프로필 관리

**User Story:** 부모 또는 아이로서, 나는 개인 프로필을 만들고 선택할 수 있어야 한다. 그래야 각자의 학습 기록이 분리되어 관리된다.

#### 인수 기준

1. WHEN 앱을 처음 실행하면 THEN 프로필 생성 화면이 표시되어야 한다.
2. WHEN 프로필을 생성할 때 THE 시스템은 이름(필수)과 학년(초등학교 5학년 / 중학교 2학년)을 입력받아야 한다.
3. WHEN 프로필 생성 시 THE 시스템은 아바타 이미지를 선택할 수 있는 옵션을 제공해야 한다.
4. IF 프로필이 이미 5개 존재하면 THEN 새 프로필 생성 버튼이 비활성화되어야 한다.
5. WHEN 프로필을 선택하면 THEN 해당 프로필의 마지막 활동일이 업데이트되어야 한다.
6. WHEN 프로필을 삭제하면 THEN 해당 프로필의 모든 학습 기록(세션, 결과, 배지)도 함께 삭제되어야 한다.
7. THE 시스템은 최대 5개의 프로필을 지원해야 한다.
8. WHEN 쿠키가 프로필 선택 화면에 표시될 때 THEN 쿠키는 신남(🐾) 감정으로 인사 메시지를 표시해야 한다.

### 요구사항 2: 쿠키 캐릭터 시스템

**User Story:** 아이로서, 나는 강아지 쿠키와 대화하듯 공부하고 싶다. 그래야 공부가 재미있고 쿠키의 반응을 보며 동기부여가 된다.

#### 인수 기준

1. THE 시스템은 쿠키 캐릭터를 모든 학습 화면에 말풍선과 함께 표시해야 한다.
2. THE 시스템은 쿠키의 감정 상태를 5가지로 구분해야 한다: 기본(😊), 신남(🐾), 칭찬(🎉), 고민(🤔), 위로(🥺).
3. WHEN 학습 단계가 변경될 때 THEN 쿠키의 감정 상태가 자동으로 전환되어야 한다.
   - 인사/학습 시작: 신남(🐾)
   - Quiz 출제/힌트: 고민(🤔)
   - 정답: 칭찬(🎉)
   - 오답: 위로(🥺)
   - 세션 완료: 칭찬(🎉)
4. WHEN 쿠키가 메시지를 표시할 때 THEN 0.5초 타이핑 인디케이터 후 메시지가 나타나야 한다.
5. THE 시스템은 인사, 대기, 힌트 전 멘트 등 반복 상황에서 로컬 템플릿 메시지 풀에서 랜덤으로 선택해야 한다 (API 호출 없음).
6. WHEN AI가 설명·Quiz·힌트·피드백을 생성할 때 THEN 쿠키 페르소나 시스템 프롬프트가 적용되어 쿠키의 말투로 출력되어야 한다.
7. THE 시스템은 쿠키 캐릭터 이미지를 감정 상태별로 5종 제공해야 한다 (Assets에 `cookie_normal`, `cookie_excited`, `cookie_praising`, `cookie_thinking`, `cookie_comforting`).
8. WHEN 쿠키 캐릭터 이미지가 표시될 때 THEN 부드러운 상하 반복 애니메이션이 적용되어야 한다.
9. WHERE 쿠키 말풍선이 표시될 때 THE 시스템은 말풍선 꼬리 모양을 포함한 둥근 카드 형태로 표시해야 한다.

### 요구사항 3: API Key 설정 및 보안

**User Story:** 부모로서, 나는 Gemini API Key를 안전하게 설정하고 싶다. 그래야 아이들이 AI 기능을 사용할 수 있고 키가 외부에 노출되지 않는다.

#### 인수 기준

1. WHEN 앱을 실행할 때 Keychain에 API Key가 없으면 THEN 설정 화면으로 이동하여 API Key 입력을 유도해야 한다.
2. WHEN API Key를 입력할 때 THE 시스템은 "AIza"로 시작하는 39자 형식인지 실시간으로 검증해야 한다.
3. WHEN API Key 형식이 올바르지 않으면 THEN 저장 버튼이 비활성화되어야 한다.
4. WHEN API Key를 저장할 때 THE 시스템은 iOS Keychain(`kSecAttrAccessibleWhenUnlockedThisDeviceOnly`)에 저장해야 한다.
5. THE 시스템은 API Key를 UserDefaults, CoreData, 로그에 저장하거나 출력해서는 안 된다.
6. WHEN API Key를 삭제할 때 THE 시스템은 Keychain에서 완전히 제거해야 한다.
7. THE 시스템은 API Key 입력 필드에 마스킹(●●●●) 옵션을 제공해야 한다.

### 요구사항 4: AI 연동 (Gemini API)

**User Story:** 아이로서, 나는 쿠키가 내 학년 수준에 맞게 설명하고 문제를 내줬으면 한다. 그래야 너무 어렵거나 쉽지 않게 공부할 수 있다.

#### 인수 기준

1. THE 시스템은 `AITutorProtocol`을 통해 AI 제공자를 추상화해야 한다 (Gemini → 다른 AI로 교체 가능).
2. WHEN 학습 설명을 생성할 때 THE 시스템은 500자 이하로 제한하고 쿠키 말투로 출력해야 한다.
3. WHEN Quiz를 생성할 때 THE 시스템은 3개 이상 10개 이하의 주관식 서술형 문제를 생성해야 한다.
4. IF 요청된 Quiz 수가 3 미만이면 THEN 3개로, 10 초과이면 THEN 10개로 자동 조정해야 한다.
5. WHEN Quiz를 생성할 때 THE 시스템은 문제 안에 정답을 포함해서는 안 된다.
6. WHEN 답변을 평가할 때 THE 시스템은 핵심 키워드 포함 여부를 기준으로 0~100점 점수를 산출해야 한다.
7. WHEN 피드백을 생성할 때 THE 시스템은 정답이면 쿠키가 크게 칭찬하고, 오답이면 따뜻하게 설명하는 말투로 출력해야 한다.
8. WHEN 힌트를 생성할 때 THE 시스템은 정답을 직접 알려주지 않아야 한다.
9. THE 시스템은 학년별 어휘 수준을 분리해야 한다: 초등학교 5학년은 한자어 최소화·짧은 문장, 중학교 2학년은 교과서 용어 포함.
10. WHEN Gemini API 요청이 실패할 때 THE 시스템은 지수 백오프(1초→2초→4초) 방식으로 최대 3회 재시도해야 한다.
11. THE 시스템은 분당 15회, 일 1,500회 API 요청 한도를 추적하고 초과 시 적절한 오류를 발생시켜야 한다.

### 요구사항 5: 대화형 학습 세션

**User Story:** 아이로서, 나는 쿠키와 대화하듯 공부하고 싶다. 그래야 단순히 문제를 푸는 것이 아니라 쿠키와 소통하며 즐겁게 학습할 수 있다.

#### 인수 기준

1. WHEN 학습을 시작할 때 THEN 쿠키가 신남(🐾) 감정으로 인사 메시지를 먼저 표시해야 한다.
2. THE 시스템은 과목(수학, 영어, 과학, 국어)과 난이도(쉬움, 보통, 어려움)를 선택할 수 있어야 한다.
3. WHEN 학습이 시작되면 THEN 쿠키가 학습 주제를 설명하고 이어서 Quiz를 출제해야 한다.
4. WHEN Quiz가 표시될 때 THEN 쿠키의 말투("자, 문제야! ... 잘 생각해봐~ 🤔")로 문제가 표시되어야 한다.
5. WHEN 아이가 답변을 입력하는 동안 THEN 쿠키는 대기 메시지("천천히 생각해도 돼. 쿠키가 기다릴게! 🐶")를 표시해야 한다.
6. THE 시스템은 답변 입력을 최대 1,000자로 제한해야 한다.
7. IF 답변이 1,000자를 초과하면 THEN 제출 버튼이 비활성화되어야 한다.
8. WHEN 답변을 제출하면 THEN 쿠키가 정답/오답 여부에 따라 감정을 전환하고 피드백을 표시해야 한다.
9. WHEN 힌트를 요청할 때 THEN 쿠키가 먼저 힌트 전 멘트를 표시한 후 AI 힌트를 전달해야 한다.
10. THE 시스템은 힌트를 최대 3단계까지 제공해야 하며, 3단계 이후 힌트 버튼은 비활성화되어야 한다.
11. WHEN 모든 Quiz를 완료하면 THEN 쿠키가 칭찬(🎉) 감정으로 세션 완료 메시지를 표시하고 정답률을 보여줘야 한다.
12. WHEN 세션이 완료되면 THE 시스템은 학습 결과를 CoreData에 저장해야 한다.

### 요구사항 6: YouTube 연계 학습

**User Story:** 아이로서, 나는 좋아하는 YouTube 영상을 보면서 쿠키와 함께 공부하고 싶다. 그래야 영상 내용을 바탕으로 더 재미있게 학습할 수 있다.

#### 인수 기준

1. THE 시스템은 YouTube URL(youtube.com/watch?v=, youtu.be/, youtube.com/embed/ 형식)을 입력받아 영상 ID를 추출해야 한다.
2. WHEN YouTube 영상이 연계되면 THEN iPad 화면을 좌측 50%(영상) + 우측 50%(쿠키 학습 패널)로 분할해야 한다.
3. THE 시스템은 WKWebView를 사용하여 YouTube 영상을 인라인으로 재생해야 한다.
4. WHEN 영상 URL이 입력되면 THEN 쿠키가 영상 제목을 기반으로 학습 주제를 자동 추출하고 표시해야 한다.
5. WHEN 영상 연계 학습을 시작하면 THEN 이후 학습 흐름은 일반 대화형 학습 세션과 동일하게 진행되어야 한다.
6. WHEN 쿠키가 YouTube 연계 화면에 표시될 때 THEN 우측 패널에 쿠키 말풍선이 표시되어야 한다.

### 요구사항 7: 학습 진행 현황 (Progress)

**User Story:** 아이와 부모로서, 나는 학습 진행 상황을 한눈에 보고 싶다. 그래야 어떤 과목을 잘하고 어떤 과목을 더 공부해야 하는지 알 수 있다.

#### 인수 기준

1. THE 시스템은 세션별 정답률(0.0~1.0)을 CoreData에 저장해야 한다.
2. THE 시스템은 연속 학습 스트릭(현재 스트릭, 최장 스트릭)을 추적해야 한다.
3. WHEN 하루에 한 번 이상 학습 세션을 완료하면 THEN 스트릭이 1 증가해야 한다.
4. THE 시스템은 과목별 평균 정답률과 총 세션 수를 표시해야 한다.
5. WHEN 특정 과목의 정답률이 80% 이상이 되면 THEN 해당 과목 성취 배지를 자동으로 부여해야 한다.
6. THE 시스템은 최근 학습 세션 목록을 과목별로 필터링하여 표시할 수 있어야 한다.
7. THE 시스템은 누적 학습 시간(분)을 프로필별로 추적해야 한다.

### 요구사항 8: 에러 처리 및 오프라인 대응

**User Story:** 아이로서, 나는 인터넷이 불안정하거나 오류가 생겨도 쿠키가 친근하게 알려줬으면 한다. 그래야 당황하지 않고 어떻게 해야 할지 알 수 있다.

#### 인수 기준

1. WHEN 네트워크 연결이 없을 때 THEN 쿠키가 위로(🥺) 감정으로 오프라인 안내 메시지를 표시해야 한다.
2. WHEN API 분당 요청 한도(15회)를 초과하면 THEN 쿠키가 "잠시 후 다시 시도해요" 메시지를 표시하고 60초 후 자동 재시도해야 한다.
3. WHEN 일일 API 요청 한도(1,500회)를 초과하면 THEN 쿠키가 "오늘 학습 한도에 도달했어요. 내일 다시 만나요!" 메시지를 표시해야 한다.
4. WHEN API Key가 유효하지 않으면 THEN 설정 화면으로 이동하는 버튼과 함께 안내 메시지를 표시해야 한다.
5. WHEN 학습 세션 중 오프라인으로 전환되면 THEN 현재 Quiz까지는 완료를 허용하고 다음 AI 요청 시 오프라인 안내를 표시해야 한다.
6. WHEN CoreData 저장에 실패하면 THEN 쿠키가 "학습 기록 저장에 실패했어요" 메시지를 표시하고 재시도 버튼을 제공해야 한다.
7. THE 시스템은 NWPathMonitor를 사용하여 네트워크 상태를 실시간으로 모니터링해야 한다.

## 정확성 속성 (Correctness Properties)

아래 속성들은 어떤 입력값이 들어와도 반드시 만족해야 하는 불변 조건이다.

| # | 속성 | 조건 |
|---|------|------|
| CP-1 | Quiz 수 범위 | 요청된 Quiz 수에 관계없이 실제 생성 수는 항상 3 이상 10 이하 |
| CP-2 | 힌트 단계 범위 | 힌트 단계는 항상 1 이상 3 이하 |
| CP-3 | 답변 길이 제한 | AI에 전달되는 답변은 항상 1,000자 이하 |
| CP-4 | 설명 길이 제한 | AI가 생성한 설명은 항상 500자 이하 |
| CP-5 | 프로필 수 제한 | 프로필은 항상 5개 이하 |
| CP-6 | 힌트 정답 미포함 | 힌트 텍스트에 정답이 직접 포함되지 않음 (프롬프트 설계로 보장) |
| CP-7 | 정답률 범위 | 세션 정답률은 항상 0.0 이상 1.0 이하 |
| CP-8 | API Key 격리 | API Key는 Keychain에만 저장되며 UserDefaults·CoreData·로그에 저장되지 않음 |
| CP-9 | 학년별 어휘 분리 | 모든 AI 프롬프트에 학년별 어휘 수준 지시가 포함됨 |
| CP-10 | 세션 완료 저장 | 세션 완료 시 반드시 CoreData에 저장되고 스트릭이 업데이트됨 |
